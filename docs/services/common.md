# NovaChat 公共类型与常量定义 (`common.h`)

## 技术职责

`common.h` 是整个 NovaChat 服务端代码库的**基础头文件**，它定义了所有 C++ 微服务共享的核心类型、常量和内联工具函数。该文件不依赖任何外部库，仅使用 C++ 标准库 (`cstdint` / `string` / `chrono` 等)。

主要内容包括：

- **命名空间 `nova`**：所有服务端代码统一使用的 C++ 命名空间，避免符号冲突。
- **版本号 `kVersion`**：标记当前代码基线为 `0.1.0`。
- **时间类型 `Timestamp`**：定义为 `int64_t`，以毫秒为单位的 Unix 时间戳，在整个系统中作为时间基线统一使用。
- **业务常量**：涵盖用户名长度限制 (`3–32`)、密码长度 (`8–128`)、消息最大字符数 (`4096`)、批量操作上限 (`100/500`)、慢速模式范围等。
- **Snowflake 参数**：定义 ID 生成器的纪元 (`2024-01-01`)、Worker ID 位数 (`10`，支持 1024 个节点)、序列号位数 (`12`，每毫秒 4096 个 ID)。
- **Token TTL**：访问令牌 1 小时、刷新令牌 30 天，控制认证有效期。
- **在线路由参数**：Redis 会话路由 TTL (30 秒)、网关心跳间隔 (15 秒)，用于实时路由刷新。
- **内联函数 `NowMs()`**：获取当前毫秒级 Unix 时间戳，为 Snowflake 生成器和各种超时逻辑提供时间基准。

## 业务角色

在 NovaChat 即时通讯系统中，`common.h` 扮演了**全局契约**的角色。它确保所有微服务——无论是用户服务、消息服务还是网关——在常量定义和类型系统上保持一致。例如，所有服务使用同一个 `Timestamp` 类型处理时间，使用同一组 Snowflake 参数来生成和解析 ID。

业务层面的约束（用户名长度、消息长度、密码复杂度等）被集中定义在此处，使得校验逻辑在整个后端系统中保持一致，不会出现不同服务对同一字段有不同限制的 bug。

## 系统连接

- **被所有 C++ 微服务引用**：`config` / `logger` / `snowflake` / `mysql_pool` / `redis_client` 等都间接或直接依赖于 `common.h`。
- **Snowflake ID 生成器**使用 `kSnowflakeEpoch` / `kWorkerIdBits` / `kSequenceBits` 等常量来构造 64 位唯一 ID。
- **网关服务**使用 `kHeartbeatInterval` / `kSessionRouteTTL` 来管理长连接心跳和路由表刷新。
- **用户服务 / 消息服务**使用 `kMaxUsernameLen` / `kMaxMessageLen` 等进行输入校验。该文件是整个服务端代码的编译依赖根节点。
